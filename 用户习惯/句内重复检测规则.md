# 句内重复检测规则

## 定义

句内重复是指说话者在一句话中说出了相同或相似的句子开头，表现为开头部分重复而后半部分不同或相反。

## 判断方法

### 核心标准

| 标准 | 说明 |
|------|------|
| 重复位置 | 句子开头 |
| 重复长度 | ≥5个相同字符 |
| 重复间隔 | 相邻句子之间 |
| 语义关系 | 后者更完整或更准确 |

### 判断示例

```
符合重复:
"你再关你关掉" - "你再关"被重复
"它是它不是" - "它是"被重复
"这个这个这个很" - 连续重复开头

不符合重复:
"你知道你知道吗" - 虽然开头相同，但这是反问强调
"然后然后然后" - 这是连续语气词，不是句内重复
```

## 重复类型

### 1. 部分重复（最常见）
```
原: "你再关[静音]你关掉"
分析: "你再关" 被截断后重新说 "你关掉"
处理: 删除 "你再关[静音]"
```

### 2. 否定纠正
```
原: "它是它不是"
分析: 前半句被否定纠正
处理: 删除 "它是"，保留 "它不是"
```

### 3. 补充纠正
```
原: "我叫张三我叫李四"
分析: 后者更正了前者
处理: 删除 "我叫张三"，保留 "我叫李四"
```

### 4. 强调重复（不删除）
```
原: "不不不是这样"
分析: "不"是强调否定，非重复
处理: 保留
```

## 重复判断矩阵

| 前句状态 | 后句关系 | 处理 |
|----------|----------|------|
| 被静音截断 | 后句完整 | 删前 |
| 语义相反 | 否定关系 | 删前 |
| 语义相近 | 补充关系 | 删较短的 |
| 语义不同 | 独立关系 | 不删 |
| 强调词开头 | 反问强调 | 不删 |

## 处理规则

### 删前保后原则

**默认**：后说的更完整，删除前面的

```
原: "你再关[静]你关掉"
处理: 删 "你再关[静]" → "你关掉"
```

### 特殊情况

1. **用户明确偏好**：`前面更顺` → 删后保前
2. **关键信息在前** → 保留前者
3. **前后矛盾** → 保留后者

## 检测算法

### 步骤

1. **句子切分** - 按静音或标点切分句子
2. **开头提取** - 提取每句前10字
3. **相似度计算** - 计算相邻句子开头相似度
4. **重复判定** - 相似度 >0.6 且有截断特征 → 判定为重复

### 截断特征

- 中间有静音（0.3s+）
- 重复词边界不完整
- 后句更完整流畅

## 句间重复 vs 句内重复

| 类型 | 特征 | 处理 |
|------|------|------|
| 句内重复 | 同一句子内重复开头 | 删前面的 |
| 句间重复 | 两个完整句子重复 | 删较短的句子 |

### 句间重复示例
```
原: "我认为这个是对的。我认为这个是对的。"
分析: 两个完整句子相同
处理: 删除第一个完整的
```

## 输出格式

检测结果输出：
```json
{
  "type": "repeated_phrase",
  "start": 12.5,
  "end": 15.8,
  "before": "你再关",
  "after": "你关掉",
  "action": "delete_before",
  "confidence": 0.95
}
```

## 用户偏好

- `删前保后` - 默认策略
- `删后保前` - 用户认为前面更顺
- `遇到重复全删` - 不保留任何重复
- `遇到重复全留` - 不自动删除任何重复
