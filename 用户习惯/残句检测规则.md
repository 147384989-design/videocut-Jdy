# 残句检测规则

## 定义

残句是指说话者在表达过程中被打断或自然中止，留下不完整的句子。

## 残句特征

| 特征 | 说明 |
|------|------|
| 不完整 | 句子明显缺少谓语或宾语 |
| 被截断 | 被静音、停顿或后续句子打断 |
| 无意义 | 残句部分无法独立表达完整意思 |

## 残句类型

### 1. 静音截断型
```
原: "我觉得这个[静音1.5秒]后来再处理"
特征: 残句后有静音
处理: 删除整个残句
```

### 2. 自我纠正型
```
原: "我叫张[停顿]我姓李"
特征: 残句后有完整纠正
处理: 删除残句，保留纠正后
```

### 3. 被打断型
```
原: "我想跟你说[被他人打断]..."
特征: 残句后有外部打断
处理: 删除残句
```

### 4. 话题转移型
```
原: "关于这个问题[静音]我们看看下一个"
特征: 残句后直接转移话题
处理: 删除残句
```

## 判断标准

### 完整性判断

| 检查项 | 说明 |
|--------|------|
| 是否有主语 | 无主语 → 可能是残句 |
| 是否有谓语 | 无谓语 → 残句 |
| 是否完整 | 语义不完整 → 残句 |
| 是否有下文 | 有完整下文补充 → 删残句 |

### 残句标记条件

必须满足以下条件之一：
1. 句子以主语开头但无谓语
2. 句子以连词开头（然后、但是、因为）
3. 句子以代词开头但无后续
4. 句子明显不完整且后续有静音

### 非残句情况

以下情况不视为残句：
- 完整的短句（如"好的"、"可以"）
- 感叹句（如"太好了！"）
- 疑问句（如"怎么办？"）
- 命令句（如"开始吧"）

## 处理规则

### 核心原则

**删前保后**：残句后面通常有更完整的表达

```
原: "关于这个[静音]我们看下一页"
处理: 删除 "关于这个" → "我们看下一页"
```

### 特殊情况

| 情况 | 处理 |
|------|------|
| 残句包含关键信息 | 保留并标记 |
| 残句后是长停顿 | 删除残句 |
| 残句后有纠正 | 删除残句，保留纠正 |
| 残句后无下文 | 删除残句 |

## 连续残句检测

### 多个连续残句
```
原: "关于[静音]这个[静音]问题[静音]我们..."
处理: 删除所有连续残句 → "我们..."
```

### 残句与重复词结合
```
原: "这个这个[静音]我觉得"
处理: 删除卡顿词和残句 → "我觉得"
```

## 与其他规则的关系

### 残句 vs 静音

| 场景 | 判断 | 处理 |
|------|------|------|
| 静音前是残句 | 残句检测 | 删除残句+静音 |
| 静音前是完整句 | 静音检测 | 仅处理静音 |
| 静音在句尾 | 自然停顿 | 保留 |

### 残句 vs 重复

| 场景 | 判断 | 处理 |
|------|------|------|
| 残句+重复词 | 先删重复词 | 再判残句 |
| 重复后残句 | 重复检测 | 删除整个重复 |
| 残句后纠正 | 残句检测 | 删除残句 |

## 检测算法

### 步骤

1. **句子切分** - 按静音和标点切分
2. **完整性检查** - 判断每个句子是否完整
3. **后续判断** - 检查残句后是否有完整内容
4. **综合判定** - 结合上下文判断是否残句

### 完整性评分

```
评分维度:
- 主语(1分): 有主语
- 谓语(2分): 有谓语
- 宾语(1分): 有宾语
- 独立完整(1分): 可独立成句

总分 < 3分 → 判定为残句
```

## 输出格式

```json
{
  "type": "incomplete_sentence",
  "start": 8.5,
  "end": 10.2,
  "content": "关于这个",
  "reason": "无谓语，被静音截断",
  "next_content": "我们看下一页",
  "action": "delete",
  "confidence": 0.88
}
```

## 用户偏好

- `残句全删` - 删除所有残句
- `保留含关键词的残句` - 残句含重要信息时保留
- `残句阈值调高` - 提高完整性判定标准
